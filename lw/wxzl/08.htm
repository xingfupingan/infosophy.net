三、自由许可协议

什么是左版？  

理查德・斯多尔曼

使一个程序成为自由程序的最简单方法是把它无版权地放到公共领域(译者注：public domain，公共领域，参见“自由和非自由软件分类”中的“公域软件”条)。这样可以让人们共享程序和对程序进行的改进，如果他们确实很介意这一点的话。但这样也会方便那些非合作者把程序变成私有软件。他们可以对（自由）程序进行或大或小的改进，然后作为私有软件的产品。这样一来，购买这些软件的人就不再享有这些软件的原创作者赋予软件的自由属性，中间人剥夺了这种自由属性。

GNU工程的目标是让所有用户都有再发布和修改GNU软件的自由。如果中间人剥夺了这种自由，我们可能还会有许多用户，但他们将没有了应有的自由。因此，为了防止这种情况，我们不是把GNU程序发布到公共领域，而是使这些程序左版化，让人们自由地“左版”（Copyleft）化GNU程序。左版，就是说，任何一个再次发布（自由）软件的人，不管他是否对该软件进行了改进，都必须使拷贝和改进软件的自由能够继续存在。左版原则保证每一个用户的自由。  

左版也激励其他编程员为自由软件作出贡献。例如象GNU C++编译器这样的重要自由程序，就是仅仅因为这些人才得以存在。  

左版原则也为那些希望对自由软件的改进出力的编程员提供一种许可。这些编程员为了赚更多的钱，愿意为公司和学校编制任何程序。一个编程员也许希望把她自己的改编贡献给（GNU）社域，但其公司却希望把她所改进了的部分转化为私权软件产品。我们向他们的老板解释，如果不把改进了的版本作为自由软件来发布，这个发布就是非法的。这时，这个老板通常就会决定把这些部分作为自由软件发布而不是把它们扔掉。

要使一个程序成为左版的，首先需要使它具有版权；然后我们附上发布规定，使这些规定成为法律武器，好让每个人有权使用、改进和再发布程序的源代码，或其他源于该程序的程序，但条件是上述发布规定（本身）不能被改变。这样，代码和自由就成为在法律上不可分割的了。

私权软件开发者用版权来夺走用户的自由，我们则用版权来保卫他们的自由。这就是为什么我们要颠倒名称，把“版权”（copyright）改变成“左版”(copyleft)的原因。

左版是一个一般概念，其细节需要从多方面加以说明。在GNU 工程中，特定的发布规定请参见我们的GNU GPL（GNU 通用公共许可规定，GNU General Public License）。而GNU LGPL（GNU 简化的通用公共许可规定,GNU Lesser General Public License），作为一种替换的形式，适用于少数（不是全部）GNU函数库。这个许可规定以前称为Library GPL, 但是我们改了名称，因为这个旧名称鼓励频繁使用这个许可证，而这个许可证不应该如此多地被使用。如果需要了解为什么这个名称的改变是必要的，请参看“为什么不要把GNU 库通用许可协议（GNU Library GPL） 作为你的第二个库项目？”（译注：参见网页：http://www.gnu.org/licenses/why-not-lgpl.html ）。  

虽然GNU 库通用许可协议（GNU Library General Public License） 已经正式被上述许可规定取代了，但它还是有效的。  

在许多手册和每个GNU 源代码的发布中，都写有这些适当的许可规定。  

GNU通用公共许可协议制定出来就是要使你能够容易地把它用于你拥有版权的程序。你不需要通过修改GNU通用公共许可协议的方式来作到这一点，你只需要在你的程序中增加一些说明，而你的程序与GNU GPL 是一致的。  

如果你愿意根据GNU GPL 许可规定对你的程序进行左版，请参看GPL 许可文本末尾处的说明。  

用同样的发布条款来发布许多不同的程序很有好处，可以使你在多种不同程序之间拷贝代码。既然遵行的是同样的发布条款，就不必考虑条款的兼容问题。简写版的通用公共许可规定包括这样一个规定，它允许你把（你的）发布条款改写成为通用公共许可规定，这样，你就可以把代码拷进一个符合通用公共许可规定的程序中去。  

关于GPL 的翻译版本（译注：以下文字摘自GNU GPL 末尾处关于GPL 的外语文字翻译版本的说明）

此版本是非正式的。从法律上讲，最初的GPL版本（英文版）专门为GNU程序规定一些实际的发布条款。

自由软件基金不把这些译文规定为正式合法译文的理由是，对这些译文的检验既困难又昂贵（需要有其它国家的双语律师的帮助）。更为不好的是，如果真的漏掉了一个错误，结果那就会给整个自由软件共同体造成惨重损失。只要译文是非正式的（即没有法律效力―译者注），它们就不会造成任何危害。但我们希望这些译文能够帮助更多的人理解GPL。

我们允许把GPL或LGPL翻译成其它语言，条件是：（1）你得声明你的译文是非正式的（下面将说明如何声明），让别人知道这不是法律意义上的正式译文；（2）如果我们从其他GNU的朋友那里得知有更好的译法，你将同意我们对译文进行修改的要求。

声明你的译文是非正式的译文的方法是，以英语和译文所使用的语言在译文前面加上以下文字（以译文使用的语言名替换language这个词）：

This is an unofficial translation of the GNU General Public License into language. It was not published by the Free Software Foundation, and does not legally state the distribution terms for software that uses the GNU GPL--only the original English text of the GNU GPL does that. However, we hope that this translation will help language speakers understand the GNU GPL better.

（汉语版声明样本）

这是非正式的、汉语版的GNU 通用公共许可协议。这个版本不是由自由软件基金会发行的，不具有对归属于GNU通用公共许可协议的软件发布的法律声明地位――只有英文版GNU通用公共许可协议才具有这样的法律地位。但我们希望这个译本能帮助讲汉语的使用者更好地理解GNU 通用公共许可协议。
